## 1. 当前领域的痛点 (Pain Points)

### 1.1 SAM 类模型在医学影像中的“水土不服”与交互负担
- **原生 SAM 不支持文本 Prompt**：原生 SAM-2 仅支持视觉 Prompt（点、框、掩码），缺乏内置的处理和理解文本语义提示的能力，这限制了其在需要细粒度上下文指导的场景中的应用。
- **交互负担过重 (High user interaction burden)**：现有的 SAM 衍生医学大模型（如 MedSAM, SegVol）在处理 3D 医疗体积数据时，要么执行 2D 逐切片分割（忽略了 3D 层间连续性），需要密集的切片级交互；要么直接 3D 化导致参数爆炸和细粒度丢失。同时，原生 SAM 依赖大量的手动专家提示，这在临床上极其耗时且容易出错。
- **多目标场景下的语义区分无力**：传统的交互式 SAM 只能依赖类别不可知（class-agnostic）的点/框提示，在多目标重叠或形态相似的场景下，无法仅通过交互提示实现对不同解剖结构的语义区分。

### 1.2 跨模态对齐的“粒度鸿沟”
- **抽象语义与像素级边界的错位**：VLM 的语义表示本质上是全局的、非像素级的，而医学分割需要精确的像素级边界定位（特别是对于胰腺、胆囊等复杂小器官）。粗暴地融合这些未对齐的特征会模糊细粒度细节并破坏分割精度。
- **通用文本提示的领域差距**：现有基于 VLM 的方法依赖于粗粒度的通用文本，缺乏临床特异性的解剖描述，导致自然图像预训练的 VLM 与医学图像之间存在严重的领域差距。

## 2. 有价值的闪光点 (Valuable Innovations/Ideas)

### 2.1 创新的 3D/视频 连续性建模 (Propagation Paradigm)
- **基于传播的 3D 建模 (Propagation-based framework)**：《PAM》提出了一种“单一切片提示 -> 沿Z轴双向传播”的范式。通过混合 CNN-Transformer 架构（交叉注意力机制）对相邻切片进行长程层间结构和语义关系建模。这种设计打破了 2D SAM 处理 3D 数据时的高交互负担，仅需一个 2D Prompt 就能自动重建 3D 目标，极大地降低了临床操作成本。
- **文本引导的内存追踪 (Text-Tracking Memory Encoder)**：《TGSAM-2》针对 SAM-2 的内存编码器进行了改造，将文本特征作为稳定的语义先验加入到历史视觉特征和预测掩码中。这解决了医疗视频/3D数据在低对比度、模糊场景下目标跟踪丢失的问题，确保了跨帧分割的一致性。

### 2.2 离线文本先验的深度应用 (Prompt-free & Semantic-aware)
- **大模型生成领域特定描述 (Medical LLM Integration)**：《PG-SAM》在无需人工标注文本的情况下，利用专用的医学 LLM 自动为目标器官生成解剖学上精确、具有临床特异性的细粒度文本描述。这解决了 VLM 在医学领域的文本粒度限制。
- **无需视觉提示的 Prompt-Free 设计**：《PG-SAM》通过将 LLM 生成的细粒度文本先验转化为语义指导矩阵（Semantic Guide Matrix G），并与视觉特征融合，成功实现了一个完全无需人工视觉 Prompt（点、框）的统一多器官分割管道。

### 2.3 细粒度特征融合与解码器优化
- **文本条件的视觉感知 (TCVP)**：《TGSAM-2》通过多头交叉注意力，使用文本特征作为 Query 来引导 SAM-2 的深层视觉特征，使其具备了医学领域的语义感知能力。
- **形变卷积解决空间未对齐**：《PG-SAM》在解码阶段引入形变卷积（Deformable Convolution），以实现语义指导矩阵与上采样视觉特征之间的精确空间对齐，捕获粗糙特征中丢失的互补边缘和轮廓线索。
- **迭代掩码优化 (Iterative Mask Optimizer)**：《PG-SAM》设计了基于超网络的实例自适应动态卷积核生成机制，并采用渐进式残差细化策略来迭代纠正掩码预测。这极大地改善了初始掩码预测边缘粗糙的问题。
