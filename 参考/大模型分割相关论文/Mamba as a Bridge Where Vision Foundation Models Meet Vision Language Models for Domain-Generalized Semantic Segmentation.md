# Summary of *Mamba as a Bridge: Where Vision Foundation Models Meet Vision Language Models for Domain-Generalized Semantic Segmentation*
## 1. Problem to Be Solved
This work targets **Domain-Generalized Semantic Segmentation (DGSS)**, a critical task for real-world applications (e.g., autonomous driving) that requires models to generalize to unseen target domains without access to target domain data during training. The core limitations of existing methods are as follows:
1.  **Underutilization of complementary model strengths**: Most DGSS approaches rely exclusively on either Vision Foundation Models (VFMs, e.g., DINOv2) or Vision-Language Models (VLMs, e.g., CLIP). VFMs excel at capturing fine-grained visual details but lack text-semantic alignment, while VLMs provide robust domain-invariant text alignment but suffer from coarse feature granularity and poor object localization.
2.  **Prohibitive computational cost of fusion**: Integrating VFMs and VLMs via vanilla attention mechanisms is inefficient, as concatenating features from dual encoders doubles the patch token sequence length, leading to quadratic computational complexity for long-sequence modeling.
3.  **Unfeasible full fine-tuning and suboptimal PEFT**: Full fine-tuning of large VFM and VLM backbones incurs extreme computational overhead, while existing parameter-efficient fine-tuning (PEFT) methods fail to effectively model inter-token dependencies and cross-modal alignment between the two model families.
4.  **Poor performance of naive feature fusion**: Simple concatenation of VFM and VLM features breaks the inherent text alignment of VLM features and cannot align VFM features with text queries, resulting in worse performance than using a single model alone.

## 2. Proposed Method
The paper proposes **MFuser**, a novel Mamba-based fusion framework that efficiently unifies VFMs and VLMs for DGSS with linear scalability to sequence length. Built on a text-queried Mask2Former pipeline, MFuser consists of two core components: *MVFuser (Mamba-based Vision Fuser)* and *MTEnhancer (Mamba-based Text Enhancer)*.

### 2.1 End-to-End Pipeline
1.  **Dual Encoder Feature Extraction**: An input RGB image is fed into the frozen visual encoders of a pre-trained VFM and a CLIP-like VLM simultaneously, extracting multi-layer patch token features from both backbones.
2.  **Layer-wise Feature Fusion via MVFuser**: For each transformer block layer of the two encoders, output features from the VFM and VLM are concatenated and fed into MVFuser. The module models sequential dynamics and spatial relationships of tokens in parallel via a modified Mamba block, generates learnable feature offsets, and adds them back to the original VFM and VLM features for refinement. This enables layer-wise cross-model feature interaction while keeping backbone parameters frozen.
3.  **Text Embedding Refinement via MTEnhancer**: Fused visual features from the final encoder layer, paired with class text embeddings generated by the VLM’s frozen text encoder, are input into MTEnhancer. This hybrid attention-Mamba module first encodes inter-class relationships via self-attention, then integrates visual priors into text embeddings via a conditional Mamba block, producing image-conditioned text embeddings with enhanced cross-modal consistency.
4.  **Segmentation Decoding**: The enhanced text embeddings serve as object queries for the Mask2Former decoder, which takes multi-scale refined visual features as input and outputs the final pixel-level semantic segmentation prediction.
5.  **Training Strategy**: Only MVFuser, MTEnhancer, and the Mask2Former decoder are trainable; the entire VFM and VLM backbones (both visual and text encoders) are fully frozen to preserve their pre-trained generalization capabilities. The total loss combines the standard Mask2Former segmentation loss and a pixel-level vision-language alignment loss consistent with the adopted VLM’s pre-training objective.

### 2.2 Key Innovations (Focus on Foundation Model Utilization)
1.  **Mamba-empowered efficient VFM-VLM fusion**: This work pioneers using the State-Space Model (SSM, i.e., Mamba) as a bridge to integrate arbitrary pairs of VFMs and VLMs. This design solves the long-sequence modeling challenge caused by doubled patch tokens from dual encoders, achieving linear computational complexity with sequence length (in contrast to the quadratic complexity of attention-based fusion), and unlocks the complementary strengths of VFMs and VLMs without prohibitive overhead.
2.  **Parameter-efficient co-adapter for joint foundation model adaptation**: MVFuser is a novel Mamba-based co-adapter that enables joint fine-tuning of frozen VFM and VLM backbones. Unlike separate adapters for each model, MVFuser facilitates layer-wise cross-model feature interaction, enhancing the fine-grained granularity of VLM features with VFM priors while aligning VFM features with the text-semantic space. It outperforms convolution-based and attention-based adapters with fewer parameters and lower FLOPs, while fully preserving the pre-trained generalization ability of large foundation models.
3.  **Hybrid attention-Mamba module for cross-modal alignment**: MTEnhancer innovatively incorporates fused VFM-VLM visual priors into VLM text embeddings via a hybrid architecture, addressing the misalignment between fixed pre-trained text embeddings and downstream visual features in existing VLM-based DGSS methods. It generates image-conditioned text embeddings that serve as robust, domain-invariant queries for the segmentation decoder, significantly improving cross-modal consistency and generalization to unseen domains.
4.  **Validated VFM-VLM complementary paradigm for DGSS**: The work rigorously proves that the complementary effect between vision-only VFMs and vision-language VLMs is far more significant than that between homogeneous VLMs. It provides a lightweight paradigm for leveraging multiple large foundation models in DGSS: instead of full fine-tuning or naive ensembling, Mamba-based efficient fusion achieves state-of-the-art (SOTA) performance with only 15 hours of training on a single 24GB RTX A5000 GPU, eliminating the need for multi-GPU high-memory resources required for full fine-tuning.

## 3. Input, Output and Required Annotations
### 3.1 Input
- **Training phase**: 512×512 RGB urban scene images paired with corresponding pixel-level semantic segmentation ground truth masks.
- **Inference phase**: RGB images of arbitrary resolution from unseen target domains, with no additional input required.

### 3.2 Output
Pixel-level semantic segmentation prediction maps of the input images, with per-pixel class labels corresponding to pre-defined semantic categories (e.g., 19 classes for urban driving scenarios).

### 3.3 Required Annotations
The only required annotation for model training is the standard semantic segmentation mask label. **No additional labels or annotations are needed beyond the segmentation masks**. Specifically, the model does not require extra text annotations, domain labels, image-text pair annotations, or other manual supervision signals. Class text embeddings are directly generated from semantic category names via the frozen VLM text encoder, and the vision-language alignment loss is computed based on the VLM’s native pre-training objective with no extra annotation cost.
