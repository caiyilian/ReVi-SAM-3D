## 🔍 **问题诊断**

**MemoryAttention 的 `forward()` 从未被调用的根本原因：**

您的环境 `SAMClosedLoopEnv` 完全绕过了 SAM2VideoPredictor 的完整推理流程，转而采用了**直接调用单个 mask_decoder** 的做法：

```python
# sam_closed_loop.py 中的实现
1. _predict_mask_from_bbox() 只调用 self.mask_decoder()
2. _propagate_mask_to_adjacent_slice() 也只调用 self.mask_decoder()
3. 没有任何地方调用 SAM2VideoPredictor.track_step() 或 propagate_in_video()
```

而 **MemoryAttention 只在以下调用链中被触发**：

```
SAM2VideoPredictor.track_step()
  ↓
SAM2Base._track_step()
  ↓
SAM2Base._prepare_memory_conditioned_features()
  ↓  
self.memory_attention()  ← 您的 LLM 层和 DCNv4 都在这里！
```

**这些调用链在您当前的代码中完全没有被执行！**

## ✅ **根本解决方案**

需要重写 `SAMClosedLoopEnv` 来使用 **SAM2VideoPredictor 的完整 API**，而不是手工逐切片调用 mask_decoder：

```python
# 伪代码展示正确做法
def _predict_mask_from_bbox_correct(self, video_frames, bbox, init_slice_idx):
    # 1. 初始化推理状态
    inference_state = self.sam_predictor.init_state(video_frames)
    
    # 2. 在初始切片添加 bbox 提示
    self.sam_predictor.add_new_points_or_box(
        inference_state=inference_state,
        frame_idx=init_slice_idx,
        obj_id=1,
        box=bbox
    )
    
    # 3. 调用完整的时空追踪 API（这会激活 MemoryAttention）
    for out in self.sam_predictor.propagate_in_video(
        inference_state, 
        start_frame_idx=init_slice_idx
    ):
        # ← 这里会调用 MemoryAttention.forward()！
        pass
    
    return inference_state["output_masks"]
```

这样 `MemoryAttention` 中的 **LLM 层 + DCNv4 时空追踪器** 才会真正被激活。
